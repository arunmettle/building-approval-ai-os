# Autoresearch Loop

## Goal

Continuously reduce uncertainty in product, market, data, and implementation decisions for Building Approval AI OS.

## Research Cadence

Each loop should produce:

1. Current objective
2. Top assumptions
3. Evidence needed
4. Research tasks
5. Decisions made
6. Artifacts updated
7. Next loop focus

## Loop Stages

### 1. Frame

Define the single highest-leverage question for the current cycle.

Examples:

- Which buyer has the strongest pain and shortest sales path?
- Which councils are best suited for the first evidence pipeline?
- Which permit class has the clearest deterministic thresholds?

### 2. Gather

Collect evidence from:

- planning schemes
- council checklists and forms
- GIS and zoning data
- historical decisions
- competitor workflows
- user interviews
- certifier and installer operational patterns

### 3. Normalize

Convert findings into structured artifacts:

- jurisdiction profiles
- permit taxonomy entries
- rule candidates
- document requirement mappings
- interview insight notes
- risk and confidence assumptions

### 4. Decide

Promote only evidence-backed findings into product direction, architecture, and backlog.

### 5. Build

Implement the smallest technical change that reduces the next uncertainty.

### 6. Evaluate

Measure:

- coverage gained
- assumptions retired
- new risks surfaced
- backlog reprioritization

## Research Tracks

- Market: user pain, buyer urgency, willingness to pay
- Regulatory: rule availability, authority complexity, effective-date stability
- Data: source accessibility, quality, freshness, lineage
- Product: workflow friction, report usefulness, handoff quality
- Technical: extraction accuracy, citation validity, rules-engine coverage

## Output Rule

No important roadmap, product, or rule decision should be accepted without explicit evidence recorded in repository artifacts.
