# Research Run 001

## Objective

Turn the blueprint into an execution wedge and define the first build order.

## Key Decisions

- Start in Queensland, Australia.
- Focus on installers first, with certifier-support as the adjacent workflow.
- Focus on decks, pergolas, sheds, and patios.
- Position the product as submission-readiness and evidence-backed pre-check, not approval automation.

## Assumptions To Test

- Installers have strong willingness to pay to reduce quote friction and rejected applications.
- A small set of councils provides enough structured and semi-structured source material for a usable MVP.
- The chosen permit categories have enough repeatability for deterministic checks to carry most of the assessment.
- Human review can be reserved for high-risk and unknown cases without collapsing workflow value.

## Build Consequences

- The first technical milestone should be a deterministic assessment contract, not an LLM-heavy prototype.
- Data contracts for evidence and rule lineage are core platform primitives.
- The research loop must prioritize source accessibility and buyer pain before broader feature expansion.

## Next Questions

1. Which three councils should be targeted first?
2. What exact installer workflow should the product replace or shorten?
3. Which rule thresholds are most stable and most decision-relevant for the initial permit types?
